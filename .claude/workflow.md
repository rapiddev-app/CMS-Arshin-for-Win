# Процесс разработки ScriptManager

> Правила работы code-ассистента по итерационной разработке проекта
>
> См. также: [@tasklist.md](tasklist.md) | [@vision.md](vision.md) | [@conventions.md](conventions.md)

## Общие принципы

### Строго следовать плану
- Выполнять итерации последовательно из [@tasklist.md](tasklist.md)
- Не пропускать итерации
- Не добавлять функциональность, не указанную в плане
- Следовать принципу KISS из [@conventions.md](conventions.md)

### Итерационный подход
Каждая итерация = законченный работающий функционал, который можно протестировать

---

## Алгоритм работы над итерацией

### Шаг 1: Согласование решения
**ДО начала написания кода:**

1. Прочитать описание итерации из [@tasklist.md](tasklist.md)
2. Изучить требования из [@vision.md](vision.md)
3. Проверить соответствие [@conventions.md](conventions.md)
4. Предложить решение:
   - Какие файлы будут созданы/изменены
   - Ключевые фрагменты кода (сигнатуры функций/классов)
   - Структура данных (если применимо)
   - Подход к реализации

**Формат предложения:**
```
Итерация X: [Название]

Планирую создать:
- src/module.py - описание назначения

Ключевые компоненты:
- class ClassName: описание
- def function_name(): описание

Подход:
- краткое описание логики

Вопросы/уточнения (если есть):
- вопрос 1
- вопрос 2
```

**ЖДАТЬ ПОДТВЕРЖДЕНИЯ** перед написанием кода

---

### Шаг 2: Реализация
**После получения подтверждения:**

1. Написать код согласно утвержденному решению
2. Следовать [@conventions.md](conventions.md):
   - Комментарии на русском
   - Простые решения (KISS)
   - Обработка ошибок
   - Логирование
3. Написать код для теста из [@tasklist.md](tasklist.md)

**НЕ отклоняться от согласованного решения без повторного согласования**

---

### Шаг 3: Тестирование
**После написания кода:**

1. Показать инструкцию по тестированию (из [@tasklist.md](tasklist.md))
2. Запустить тест (если возможно автоматически)
3. Показать результаты теста

**Формат отчета:**
```
Тест итерации X:

Команда:
[команда для запуска]

Ожидаемый результат:
[что должно произойти]

Фактический результат:
[что произошло]

Статус: ✅ Успешно / ❌ Ошибка
```

**ЖДАТЬ ПОДТВЕРЖДЕНИЯ** что тест пройден

---

### Шаг 4: Обновление прогресса
**После успешного теста:**

1. Обновить таблицу прогресса в [@tasklist.md](tasklist.md):
   - Изменить статус итерации: ⬜ → ✅
   - Указать дату завершения
   - Добавить примечания (если есть)
   - Обновить процент выполнения
2. Отметить выполненные задачи чекбоксами: `- [ ]` → `- [x]`

---

### Шаг 5: Согласование перехода
**Перед началом следующей итерации:**

Спросить:
```
Итерация X завершена и протестирована.

Прогресс: X/8 (XX%)

Готов начать следующую итерацию: [Название]?
```

**ЖДАТЬ ПОДТВЕРЖДЕНИЯ** перед началом следующей итерации

## Правила работы с кодом

### Создание файлов
- Создавать только файлы, указанные в итерации
- Следовать структуре проекта из [@vision.md](vision.md)
- Использовать абсолютные пути при необходимости

### Изменение файлов
- Менять только то, что необходимо для текущей итерации
- Не рефакторить код, который работает
- Не добавлять "улучшения" без согласования

### Комментарии и документация
- Комментарии только на русском языке
- Docstrings для классов и функций
- Комментарии там, где логика неочевидна

### Обработка ошибок
Обязательно для каждой итерации:
```python
try:
    # основная логика
except Exception as e:
    logging.error(f"Ошибка: {e}")
    # send_telegram_notification если применимо
    raise
```

## Работа с зависимостями

### Если нужна новая библиотека:
1. Спросить разрешения ДО добавления
2. Объяснить зачем нужна
3. После подтверждения добавить в `requirements.txt`

### Приоритет:
1. Встроенные библиотеки Python (json, os, datetime и т.д.)
2. Уже используемые библиотеки (pandas, requests и т.д.)
3. Новые библиотеки (только с согласования)

---

## Работа с ошибками в процессе

### Если тест не прошел:
1. Показать ошибку
2. Предложить исправление
3. ЖДАТЬ ПОДТВЕРЖДЕНИЯ
4. Исправить
5. Повторить тест

### Если решение оказалось неоптимальным:
1. Объяснить проблему
2. Предложить альтернативу
3. ЖДАТЬ ПОДТВЕРЖДЕНИЯ
4. Реализовать новое решение

### Если требования неясны:
1. Задать конкретные вопросы
2. ЖДАТЬ ответа
3. Продолжить после получения ответов
