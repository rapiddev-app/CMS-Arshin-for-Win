# Автоматическое обновление производственного календаря

## 📅 Как работает автоматическое обновление

Система ScriptManager автоматически проверяет и обновляет производственный календарь РФ без участия пользователя.

### Когда происходит обновление

Проверка выполняется **при каждом запуске любого скрипта** через метод `is_workday()`.

**Условия для автоматического обновления:**

1. **Календарь устарел** (текущий год > год в календаре)
   ```
   Пример: сейчас 2026 год, а в calendar.json год 2025
   → Автоматически загружается календарь на 2026 год
   ```

2. **До конца года осталось меньше 30 дней**
   ```
   Пример: сегодня 5 декабря 2025 года
   → Автоматически загружается календарь на 2026 год
   → Система будет готова к новому году
   ```

### Откуда загружается календарь

**API:** https://isdayoff.ru/api/getdata?year=2026&cc=ru

Это официальный бесплатный API производственного календаря РФ.

**Формат данных:**
- Строка из 365 или 366 символов (для високосного года)
- `1` = выходной/праздник
- `0` = рабочий день

**Пример:**
```

│││││││││└─ 9 января (выходной)
││││││││└── 8 января (выходной)
│││││││└─── 7 января (Рождество)

││└──────── 2 января (выходной)
│└───────── 1 января (Новый год)
```

## 🔄 Процесс обновления

### Автоматическая последовательность:

```
1. Скрипт запускается (daily/weekly)
   ↓
2. Вызывается is_workday() для проверки даты
   ↓
3. check_and_update_calendar() проверяет актуальность
   ↓
4. Если нужно обновление:
   ↓
5. _update_calendar_for_year(2026) загружает новый календарь
   ↓
6. GET https://isdayoff.ru/api/getdata?year=2026&cc=ru
   ↓
7. Проверка корректности (365/366 символов)
   ↓
8. Сохранение в config/calendar.json
   ↓
9. Telegram уведомление: "📅 Календарь обновлён!"
   ↓
10. Скрипт продолжает работу с новым календарём
```

## 🛠️ Ручное обновление (если автоматическое не сработало)

### Способ 1: Через Python скрипт

```python
from src.scripts.base_script import BaseScript

script = BaseScript.__new__(BaseScript)
script.config_manager = ConfigManager()
result = script._update_calendar_for_year(2026)

if result:
    print("Календарь успешно обновлён!")
else:
    print("Ошибка обновления, см. logs/app.log")
```

### Способ 2: Вручную через API

1. Откройте браузер:
```
https://isdayoff.ru/api/getdata?year=2026&cc=ru
```

2. Скопируйте строку из 365/366 символов

3. Откройте [config/calendar.json](../config/calendar.json)

4. Обновите:
```json
{
  "year": 2026,
  "holidays": [],
  "days": "ВСТАВЬТЕ_СЮДА_СКОПИРОВАННУЮ_СТРОКУ",
  "last_update": "2025-12-02"
}
```

## ❓ Частые вопросы

### Что если API недоступен?

Система отправит уведомление в Telegram и продолжит работать:
- С существующим календарём (если год совпадает)
- По простой логике выходных (суббота/воскресенье), если год не совпадает

### Как часто проверяется календарь?

При каждом запуске скрипта через `is_workday()`.

Но загрузка происходит только при необходимости (устарел или меньше 30 дней до нового года).

### Можно ли обновить календарь заранее?

Да, можно вручную через Python скрипт или API (см. "Ручное обновление" выше).

### Что делать если обновление не сработало?

1. Проверьте логи: `type logs\app.log | findstr "календарь"`
2. Проверьте интернет-соединение
3. Проверьте доступность API: https://isdayoff.ru
4. Выполните ручное обновление (см. выше)

## 🎯 Преимущества автоматического обновления

✅ **Без участия пользователя** - всё происходит автоматически
✅ **Заблаговременно** - за 30 дней до нового года
✅ **Уведомления** - Telegram сообщения о результате
✅ **Fallback** - если не удалось, работает по простой логике выходных
✅ **Логирование** - все операции записываются в лог
