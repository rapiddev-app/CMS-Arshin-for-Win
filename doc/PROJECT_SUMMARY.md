### 2. Инфраструктура

**BaseScript (src/scripts/base_script.py)**
- Абстрактный базовый класс для всех скриптов
- Email отправка через Gmail/Yandex SMTP
- Telegram уведомления (личный чат + группа)
- Методы работы с производственным календарём РФ
- Автоматическое перепланирование в Task Scheduler
- Расчёт периода сбора данных
- Обработка ошибок и логирование

**ConfigManager (src/config_manager.py)**
- Централизованное управление конфигурацией
- Загрузка/сохранение JSON файлов
- Валидация структуры конфигов
- Создание дефолтных конфигов

**TaskSchedulerManager (src/scheduler.py)**
- Интеграция с Windows Task Scheduler через pywin32
- Создание/обновление/удаление задач
- Получение статуса задач
- Включение/отключение задач

**Runner (src/scripts/runner.py)**
- Единая точка входа для Task Scheduler
- Маппинг script_id → класс скрипта
- Обработка аргументов командной строки
- Централизованное логирование

### 3. GUI (src/gui.py)

**ScriptManagerGUI**
- Интерфейс на базе CustomTkinter
- Отображение всех 6 скриптов
- Управление скриптами через кнопки
- Интеграция с Task Scheduler
- Защита паролем (`cms2025`)

**Функционал:**
- ⚙ Настроить - редактирование всех параметров (с паролем)
- ➕ Создать задачу - создание в Task Scheduler
- ✉ Тест - немедленный запуск для проверки
- Включить/Отключить - управление активностью
- Обновить статусы - синхронизация с Task Scheduler

**PasswordDialog**
- Модальное окно для ввода пароля
- Защита критичных настроек
- Проверка пароля администратора

### 4. Конфигурация

## 📊 Workflow системы

### 1. Ежедневные скрипты (CMS 36, CMS 48)

```
09:00/10:00 → Task Scheduler запускает runner.py
            → runner.py вызывает класс скрипта
            → gather_data() - сбор данных из Excel
            → create_output_file() - создание отчёта
            → send_email() - отправка письма
            → send_telegram_notification() - уведомление
            → reschedule_next_run() - обновление Task Scheduler
            → Следующий запуск: завтра в то же время
```

### 2. Еженедельные скрипты (Kvadra, LTS, RVK, RVK Kvadra)

```
Thursday/Friday 09:00 → Task Scheduler запускает runner.py
                     → runner.py вызывает класс скрипта
                     → calculate_collection_period() - период сбора (7 дней)
                     → gather_data() - сбор данных с прошлого Thursday/Friday
                     → create_output_file() - создание отчёта
                     → send_email() - отправка письма
                     → send_telegram_notification() - уведомление
                     → reschedule_next_run() - обновление Task Scheduler
                     → Следующий запуск: следующий Thursday/Friday
```

### 3. Умное перепланирование

```
Запланирован запуск на субботу 13.12.2025
↓
smart_reschedule=True
↓
is_workday(13.12.2025) → False (суббота)
↓
get_next_workday(13.12.2025) → 15.12.2025 (понедельник)
↓
Задача переносится на 15.12.2025
```
