## Выполненные задачи

### 1. Реализованы методы работы с календарём в BaseScript ✅

Добавлены следующие методы в [src/scripts/base_script.py](../src/scripts/base_script.py):

#### `is_workday(date: datetime) -> bool`
- Проверяет, является ли дата рабочим днём
- Использует производственный календарь из [config/calendar.json](../config/calendar.json)
- Проверяет выходные (суббота=5, воскресенье=6) и праздники РФ
- Fallback на простую проверку выходных, если год не совпадает с календарём

#### `get_next_workday(date: datetime) -> datetime`
- Находит следующий рабочий день после заданной даты
- Автоматически пропускает выходные и праздники
- Защита от бесконечного цикла (максимум 10 итераций)
- Используется для "умного" переноса задач

### 2. Реализован метод расчёта следующего запуска ✅

#### `get_next_run_date(schedule_type, send_day, smart_reschedule) -> datetime`
Вычисляет следующую дату запуска с учётом типа расписания:

**Для daily скриптов:**
- Следующий день после текущей даты
- С учётом календаря (если smart_reschedule=True)
- Автоматически пропускает выходные и праздники

**Для weekly скриптов:**
- Следующий указанный день недели (например, Thursday)
- Преобразование названий дней в номера (Monday=0, Sunday=6)
- С учётом календаря (если smart_reschedule=True)

### 3. Реализован метод перепланирования задач ✅

#### `reschedule_next_run(schedule_type, send_time, send_day, smart_reschedule)`
Пересчитывает и обновляет следующий запуск после успешной отправки:

- Вычисляет следующую дату запуска через `get_next_run_date()`
- Добавляет время запуска (формат HH:MM)
- Обновляет задачу в Windows Task Scheduler через `TaskSchedulerManager.update_task_date()`
- Логирует результат операции
- Не пробрасывает ошибки (перепланирование не критично)

### 4. Улучшен метод calculate_collection_period() ✅

Уже был реализован в итерации 5, но теперь полностью интегрирован:

- Daily скрипты: собирают данные за текущий день
- Weekly скрипты: собирают данные с последнего base_weekday по actual_run_date
- Корректно работает при переносах на рабочие дни
- Пример: если четверг был праздником и перенесён на пятницу, период всё равно считается с прошлого четверга

## Примеры использования

### Daily скрипт (CMS 36 Воронеж, CMS 48 Липецк)

```python
class CMS36Script(BaseScript):
    def __init__(self):
        super().__init__('cms36')

    def run(self):
        # Основная логика выполнения
        result = super().run()

        if result:
            filename, filepath = result

            # Отправка письма
            self.send_email(filename, filepath, ...)

            # Автоматическое перепланирование на завтра (или следующий рабочий)
            self.reschedule_next_run(
                schedule_type='daily',
                send_time='10:00',
                smart_reschedule=True
            )
```

### Weekly скрипт (RVK - каждый четверг)

```python
class RVKScript(BaseScript):
    def __init__(self):
        super().__init__('rvk')

    def run(self):
        # Расчёт периода сбора с прошлого четверга
        start_date, end_date = self.calculate_collection_period(
            actual_run_date=datetime.now(),
            base_weekday=3  # Thursday
        )

        # Основная логика
        result = super().run()

        if result:
            filename, filepath = result
            self.send_email(filename, filepath, ...)

            # Перепланирование на следующий четверг
            self.reschedule_next_run(
                schedule_type='weekly',
                send_time='09:00',
                send_day='Thursday',
                smart_reschedule=True
            )
```

## Архитектурные решения

### 1. Умное перепланирование (smart_reschedule)

**Включено (по умолчанию):**
- Автоматически переносит задачи на рабочие дни
- Пропускает выходные и праздники РФ
- Подходит для большинства сценариев

**Выключено:**
- Строгое следование расписанию (daily = ежедневно, weekly = каждую неделю)
- Не учитывает календарь
- Может отправлять в выходные/праздники

### 2. Интеграция с Task Scheduler

После каждой успешной отправки:
1. Скрипт вычисляет следующую дату запуска
2. Обновляет триггер в Windows Task Scheduler
3. Task Scheduler автоматически запустит скрипт в новое время

**Преимущества:**
- Задачи сами управляют своим расписанием
- Не требуется сложная логика в Task Scheduler
- Гибкость при переносах и праздниках

### 3. Обработка ошибок

**Если перепланирование не удалось:**
- Ошибка логируется
- Задача НЕ падает (перепланирование не критично)
- Task Scheduler запустит в прежнее время через неделю/день
- Защита от дублирования через проверку существования файлов
