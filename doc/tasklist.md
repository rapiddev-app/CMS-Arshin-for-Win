# План работ по устранению замечаний и доработке (Итерация 9)

**Цель:** Исправить ошибки установки/удаления, улучшить UX интерфейса и добавить функции управления запуском.

## 1. Исправление инсталлятора и деинсталлятора

### 1.1. Исправление интерфейса `installer.py`
- **Проблема:** Кнопка "Установить" отсутствует или не видна в `ScriptManager_Setup.exe`.
- **Причина:** Фиксированный размер окна (500x400) может быть недостаточен для отображения всех элементов на экранах с масштабированием, скрывая нижнюю панель кнопок.
- **Решение:**
    - Увеличить высоту окна до 550px.
    - Разрешить изменение размера окна (`resizable(True, True)`).
    - Проверить упаковку (pack) элементов, чтобы кнопки всегда были видны (порядок упаковки: сначала кнопки снизу, потом контент).

### 1.2. Исправление деинсталляции
- **Проблема:** Удаление через "Параметры Windows" не работает.
- **Причина:** Путь к деинсталлятору в реестре может содержать пробелы (C:\Program Files\...) и не заключен в кавычки.
- **Решение:**
    - В методе `register_in_registry` добавить кавычки вокруг пути к `uninstall.bat`.
    - Проверить корректность генерации `uninstall.bat`.

## 2. Доработки интерфейса (GUI)

### 2.1. Кнопка "Запустить сейчас"
- **Задача:** Добавить возможность перезапуска/ручного запуска скрипта (не тест, а реальное выполнение).
- **Решение:**
    - Добавить кнопку "▶ Запустить" (Run) в карточку скрипта.
    - Метод запуска будет аналогичен `test_send`, но без переопределения Email получателя (использовать настройки) и с подтверждением действия.

### 2.2. Кнопка "Сохранить" в настройках
- **Проблема:** Пользователь не видит кнопку сохранения.
- **Причина:** Вероятно, та же проблема с размером окна диалога настроек (700x650) или порядком упаковки.
- **Решение:**
    - Изменить компановку диалога настроек: Кнопки "Сохранить"/"Отмена" прикрепить к нижней части окна (`pack(side="bottom")`) *до* создания прокручиваемой области. Это гарантирует, что они всегда будут поверх контента.
    - Сделать окно настроек ресайзабельным.

### 2.3. Ответ на вопрос по логике "Создать задачу" и "Включить"
- **Концепция:**
    - **"Создать задачу"**: Создает *запись* в Планировщике заданий Windows (расписание, действие). Это делается один раз.
    - **"Включить/Отключить"**: Меняет *состояние* этой задачи (Active/Disabled).
- **Связь:**
    - Кнопка "Создать задачу" создает задачу с тем состоянием (Enabled/Disabled), которое сейчас выбрано в настройках.
    - Переключение галочки "Включить" *мгновенно* обновляет состояние задачи в Windows (если она создана).
- **Действие:** Добавить поясняющую подсказку (tooltip) или улучшить описание в интерфейсе (опционально).
